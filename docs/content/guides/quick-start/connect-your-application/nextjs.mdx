## Create an Application in <ProductName />

Before integrating <ProductName /> with your Next.js app, you need to create an application in <ProductName />.

Unlike single-page apps, a Next.js application runs code on the server side. This means authentication uses a **client secret** instead of PKCE, which keeps your credentials secure server-side.

### Using the <ProductName /> Console

1. Sign into the <ProductName /> Console at [https://localhost:8090/console](https://localhost:8090/console)
2. Navigate to **Applications** → **New Application**
3. Provide the following details:
   - **Name**: `my-nextjs-app`
   - **Type**: Web Application
   - **Authorized Redirect URL**: `http://localhost:3000`

:::info
The authorized redirect URL is where <ProductName /> sends users after they successfully sign in. For this guide, use `http://localhost:3000`, the default Next.js development server URL.
:::

### Using the <ProductName /> API

You can also create an application using the <ProductName /> API. First, [obtain a system API token](/docs/next/guides/getting-started/get-thunderid#obtain-system-api-token), then:

```bash
curl -kL -X POST https://localhost:8090/applications \
  -H 'Authorization: Bearer <your-system-token>' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "my-nextjs-app",
    "inboundAuthConfig": [{
      "type": "oauth2",
      "config": {
        "grantTypes": ["authorization_code", "refresh_token"],
        "responseTypes": ["code"],
        "redirectUris": ["http://localhost:3000"],
        "tokenEndpointAuthMethod": "client_secret_post",
        "publicClient": false,
        "pkceRequired": false
      }
    }]
  }'
```

:::warning Important
Note down both the **Client ID** and **Client Secret** from the response. You'll need both to configure your Next.js app.
:::

## Create a Next.js App

Create your new Next.js app using the App Router:

<CodeGroup>
  <CodeBlock lang="bash" label="npm">
    npx create-next-app@latest my-nextjs-app --typescript --app --no-src-dir
    cd my-nextjs-app
  </CodeBlock>

  <CodeBlock lang="bash" label="Yarn">
    yarn create next-app my-nextjs-app --typescript --app --no-src-dir
    cd my-nextjs-app
  </CodeBlock>

  <CodeBlock lang="bash" label="pnpm">
    pnpm create next-app my-nextjs-app --typescript --app --no-src-dir
    cd my-nextjs-app
  </CodeBlock>
</CodeGroup>

:::info App Router required
The `@thunderid/nextjs` SDK requires the Next.js **App Router** (introduced in Next.js 13). The Pages Router is not supported.
:::

## Set Up Environment Variables

Create a `.env.local` file in the root of your project and add your <ProductName /> credentials:

```bash title=".env.local"
NEXT_PUBLIC_ASGARDEO_BASE_URL=https://localhost:8090
NEXT_PUBLIC_ASGARDEO_CLIENT_ID=<your-client-id>
ASGARDEO_CLIENT_SECRET=<your-client-secret>
```

:::warning Security
`ASGARDEO_CLIENT_SECRET` has no `NEXT_PUBLIC_` prefix — Next.js keeps it server-side only and never exposes it to the browser. Never prefix your client secret with `NEXT_PUBLIC_`.
:::

| Variable | Description |
|----------|-------------|
| `NEXT_PUBLIC_ASGARDEO_BASE_URL` | Your <ProductName /> instance URL |
| `NEXT_PUBLIC_ASGARDEO_CLIENT_ID` | The Client ID from your <ProductName /> application |
| `ASGARDEO_CLIENT_SECRET` | The Client Secret from your <ProductName /> application (server-side only) |

## Add ThunderIDProvider to Your Root Layout

The `ThunderIDProvider` is a **Server Component** that initializes authentication and makes session state available throughout your app. Wrap your root layout with it:

```tsx title="app/layout.tsx" showLineNumbers
import { ThunderIDProvider } from '@thunderid/nextjs';
import type { Metadata } from 'next';
import './globals.css';

export const metadata: Metadata = {
  title: 'My Next.js App',
};

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>
        <ThunderIDProvider
          baseUrl={process.env.NEXT_PUBLIC_ASGARDEO_BASE_URL!}
          clientId={process.env.NEXT_PUBLIC_ASGARDEO_CLIENT_ID!}
          clientSecret={process.env.ASGARDEO_CLIENT_SECRET!}
        >
          {children}
        </ThunderIDProvider>
      </body>
    </html>
  );
}
```

### Configuration Parameters

| Parameter | Description |
|-----------|-------------|
| `baseUrl` | Your <ProductName /> instance URL |
| `clientId` | The Client ID from your <ProductName /> application |
| `clientSecret` | The Client Secret (used server-side only) |
| `afterSignInUrl` | Where to redirect after sign-in (defaults to app origin) |
| `afterSignOutUrl` | Where to redirect after sign-out (defaults to app origin) |

## Add Middleware

Create a `middleware.ts` file in the root of your project. The middleware handles automatic **session refresh** so Server Components always receive a valid session:

```typescript title="middleware.ts" showLineNumbers
import { thunderIDMiddleware } from '@thunderid/nextjs/middleware';

export default thunderIDMiddleware();

export const config = {
  matcher: [
    '/((?!_next/static|_next/image|favicon.ico).*)',
  ],
};
```

:::tip Protecting Routes
You can use the middleware to require authentication on specific routes:

```typescript title="middleware.ts" showLineNumbers
import { thunderIDMiddleware, createRouteMatcher } from '@thunderid/nextjs/middleware';

const isProtectedRoute = createRouteMatcher(['/dashboard(.*)']);

export default thunderIDMiddleware(async (thunderid, req) => {
  if (isProtectedRoute(req)) {
    await thunderid.protectRoute();
  }
});

export const config = {
  matcher: ['/((?!_next/static|_next/image|favicon.ico).*)'],
};
```

Unauthenticated users visiting a protected route are automatically redirected to the sign-in page.
:::

## Add Sign-In and Sign-Out

The <ProductName /> Next.js SDK provides `SignInButton` and `SignOutButton` components along with `SignedIn` and `SignedOut` for conditional rendering. These are **Client Components** and can be used directly in your pages.

Replace the content of `app/page.tsx` with:

```tsx title="app/page.tsx" showLineNumbers
import {
  SignedIn,
  SignedOut,
  SignInButton,
  SignOutButton,
} from '@thunderid/nextjs';

export default function Home() {
  return (
    <main>
      <header>
        <h1><ProductName /> Auth Demo</h1>
        <SignedIn>
          <SignOutButton>Sign Out</SignOutButton>
        </SignedIn>
        <SignedOut>
          <SignInButton>Sign In</SignInButton>
        </SignedOut>
      </header>
    </main>
  );
}
```

## Display User Profile Information

Use the `User` component to access and display profile information for the signed-in user:

```tsx title="app/page.tsx" showLineNumbers
import {
  SignedIn,
  SignedOut,
  SignInButton,
  SignOutButton,
  User,
} from '@thunderid/nextjs';

export default function Home() {
  return (
    <main>
      <header>
        <h1><ProductName /> Auth Demo</h1>
        <SignedIn>
          <SignOutButton>Sign Out</SignOutButton>
        </SignedIn>
        <SignedOut>
          <SignInButton>Sign In</SignInButton>
        </SignedOut>
      </header>

      <section>
        <SignedIn>
          <User>
            {(user) => user && (
              <div className="user-profile">
                {user.picture && (
                  <img
                    src={user.picture}
                    alt={user.name || 'User avatar'}
                    style={{ width: '80px', height: '80px', borderRadius: '50%' }}
                  />
                )}
                <h2>Welcome, {user.name || user.username}!</h2>
                <div>
                  <p><strong>Email:</strong> {user.email}</p>
                  <p><strong>First Name:</strong> {user.given_name}</p>
                  <p><strong>Last Name:</strong> {user.family_name}</p>
                </div>
              </div>
            )}
          </User>
        </SignedIn>
      </section>
    </main>
  );
}
```
