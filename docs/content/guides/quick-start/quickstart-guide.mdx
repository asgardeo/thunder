# Quickstart Guide

Thunder is a lightweight identity platform for authentication, authorization, and user management. It supports standard protocols like OAuth 2.0 and OIDC, flexible sign-in flows, and individual mechanisms such as password, passwordless, and social login. This guide walks you through registering your first application and creating your first user so you can start testing sign-in flows right away.

:::tip Connect with AI
Install the [Thunder MCP server](/guides/mcp-server) to manage Thunder directly from your AI assistant. Create applications, configure flows, and get integration code without leaving your IDE.
:::

## Prerequisites

- Thunder is running locally or on a server. See the [Installation guide](/guides/installation) to get started.
- You have access to the Thunder Console or a system API token. To get a system API token, see [Get Thunder](/guides/quick-start/get-thunder#obtain-system-api-token).

## Step 1: Create an Application

An application in Thunder defines how your users authenticate and what they can access. Register it first to get a **Client ID** for your integration.

<Tabs>
<TabItem value="console" label="üñ•Ô∏è Console">

1. Sign into the Thunder Console at [https://localhost:8090/develop](https://localhost:8090/develop).
2. Navigate to **Applications** ‚Üí **New Application**.
3. Enter an **Application Name**.
4. Select the **Sign-In Options** ‚Äî the methods users will use to sign in (for example, Username & Password or social login).
5. Configure the **Sign-In Experience** ‚Äî select how and who can access your application:

   - **Sign-In Approach**: Choose how users access the sign-in flow.
     - **Redirect to Thunder sign-in/sign-up handling pages** (recommended) ‚Äî Thunder redirects users to system-hosted pages, which you can customize with the Flow Designer and integrate with SDKs in a few steps.
     - **Embedded sign-in/sign-up components in your app** ‚Äî users sign in directly through your app using Thunder's UI components or APIs.

   - **User Access**: Select which user types can access this application ‚Äî **Customer**, **Person**, or both.

      <details>
        <summary>User Types</summary>

        User types define the profile and attributes for different categories of users.

        Thunder includes the following default user types:

        - **Customer**: End users who interact with your application. This type allows self-registration.
        - **Person**: Administrator-managed general-purpose user accounts.

      </details>

6. Select the **Application Type** and provide a redirect URL if prompted.

    <details>
      <summary>Application Types</summary>

      | Application Type       | Description                                                                              | Required Configuration                                                                                                                                                                  |
      | ---------------------- | ---------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
      | ‚öõÔ∏è **Browser App**     | For browser-based single-page applications (React, Angular, Vue).                       | Configure an **Authorized Redirect URL** (for example, `http://localhost:3000/callback`) to receive the authentication response.                                                        |
      | üåê **Full-stack App**  | For server-rendered web applications (Express, Spring, .NET).                           | Configure an **Authorized Redirect URL** where the server handles the authentication response.                                                                                          |
      | üì± **Mobile App**      | For native or cross-platform mobile applications (Swift, Kotlin, React Native, Flutter). | Configure a **Deep Link** (for example, `com.example.app://auth/callback`) or **Universal Link** (for example, `https://app.example.com/auth/callback`) to receive the sign-in response. |
      | ü§ñ **Backend Service** | For server-to-server communication without user interaction.                             | No redirect URL required. Use the generated **Client ID** and **Client Secret** for authentication.                                                                                     |

    </details>
7. Click **Create Application**.

:::tip
Note the **Client ID** from the application details page. You need it to configure your integration.
:::

</TabItem>
<TabItem value="api" label="üß© API">

```bash
curl -kL -X POST https://localhost:8090/applications \
  -H 'Authorization: Bearer <your-system-token>' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "my-app",
    "inbound_auth_config": [{
      "type": "oauth2",
      "config": {
        "grant_types": ["authorization_code", "refresh_token"],
        "response_types": ["code"],
        "redirect_uris": ["http://localhost:5173"],
        "token_endpoint_auth_method": "none",
        "public_client": true,
        "pkce_required": true
      }
    }]
  }'
```

:::warning Important
Note the **Client ID** from the response. You need it to configure your integration.
:::

</TabItem>
<TabItem value="ai" label="ü§ñ AI Assistant">

:::info Prerequisite
[Set up the Thunder MCP server](/guides/mcp-server) and connect it to your AI assistant before following these steps.
:::

Ask your AI assistant:

> "Create a new SPA application in Thunder called `my-app` with a redirect URL of `http://localhost:5173`"

Your AI assistant returns the **Client ID** directly in the conversation.

</TabItem>
</Tabs>

## Step 2: Create a User

Create a user to test sign-in flows with your application.

<Tabs>
<TabItem value="console" label="üñ•Ô∏è Console">

1. Navigate to the **Users** section in the Thunder Console.
2. Click **Add User** and select a **User Type**.
3. Fill in the user attributes that correspond to the selected user type. For example, `username`, `email`, `given_name`, `family_name`, `password`.
4. Click **Create User**.

</TabItem>
<TabItem value="api" label="üß© API">

```bash
curl -kL -X POST https://localhost:8090/users \
  -H 'Authorization: Bearer <your-system-token>' \
  -H 'Content-Type: application/json' \
  -d '{
    "organizationUnit": "<organization-unit-id>",
    "type": "Customer",
    "attributes": {
      "username": "jane",
      "email": "jane.doe@example.com",
      "given_name": "Jane",
      "family_name": "Doe",
      "password": "Secret123!"
    }
  }'
```

Find the `<organization-unit-id>` in the Thunder Console under **Organization Units**.

</TabItem>
</Tabs>

:::info User Types & Organization Units

Each user belongs to a **user type** that determines their profile, and an **organization unit** that acts as their container.

| User Type | Description | Organization Unit |
|-----------|-------------|-------------------|
| **Customer** | Self-registering end users | Customers |
| **Person** | Administrator-managed accounts | Default |

To create custom user types, go to **User Types** in the Thunder Console.
:::
